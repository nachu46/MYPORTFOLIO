Databricks Governance & RLAC Portal

Role: Full Stack Developer

Project Overview

A premium, state-of-the-art governance administrative console and security policy management portal integrated with the Databricks Unity Catalog. This platform automates direct table-level access permissions (Grants) and implements row-level security isolation (RLAC) based on dynamic user attributes (Group, Cluster, Company, and Plant codes).

System Architecture

Backend (Node.js/Express)

  • • Role-Based Access Control: Strict middleware protection.
  • • Policy Engine: Access operations and normalizations.
  • • Databricks Driver: Direct Unity Catalog live access and policy enforcement.
  • • Data Preview: Secure query simulation engine.

Frontend (React.js SPA)

  • • Governance Console: Admin Assign Access for Grant & RLAC.
  • • Compliance Audit: Interactive audit trail logs.
  • • Employee Console: SQL Query & Access preview.
  • • Security Modals: Reusable destructive confirmation flows.

Technologies Used

Databricks SQL Unity Catalog Node.js Express.js React.js Axios

Critical Security Flows & Execution Logic

shield

1. Policy Save Flow

Strict attribute normalization prevents security bypasses. Blank fields are normalized to NULL before executing a live MERGE INTO workspace.governance.rlac_policies query.

MERGE INTO rlac_policies AS target
USING (...) AS source
ON target.user_email = source.email
visibility

2. Preview Simulation

A Risk Assessment modal safely simulates a policy before production deployment. It extracts column metadata and formulates dynamic REGEXP filtering constraints to query live data temporarily.

TRIM(group_code) REGEXP '(?i)GRP005'
lock

3. Production Enforcement

Adheres to least-privilege principles by revoking legacy access, dynamically building rlac_filter_func, and securely binding it via ALTER TABLE ... SET ROW FILTER.

ALTER TABLE hr.employee
SET ROW FILTER rlac_filter_func
