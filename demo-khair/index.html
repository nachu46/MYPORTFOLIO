About the Platform

A modern, full-stack multi-tenant Point of Sale (POS) and SaaS Billing Application. It features a React frontend and a Python (FastAPI) backend. The application supports offline-first POS operations, multi-store management, superadmin analytics, and role-based access control.

By default, web browsers force a "Print Dialog" popup. For fast POS checkout, this is bypassed via two integrated methods:

Method 1: Chrome Kiosk

Set Thermal Printer as Default. Append --kiosk-printing to the Chrome shortcut target for instant, silent hardware receipts.

Local Development Setup

1. Database Setup (Supabase / PostgreSQL)

This project uses PostgreSQL as its database. For development, you can use Supabase.

  • Create a new project in Supabase.
  • Obtain your PostgreSQL Connection String (Session mode -> port 6543).
  • The schema will be automatically initialized when the backend connects (via backend/schema.sql).

2. Backend Setup (FastAPI)

# Navigate to backend cd backend # Create and activate virtual environment (Windows) python -m venv venv .\venv\Scripts\activate # Install dependencies pip install -r requirements.txt

Create a .env file inside the backend folder:

DATABASE_URL=postgresql://postgres:[YOUR-PASSWORD]@aws-0-eu-central-1.pooler.supabase.com:6543/postgres JWT_SECRET=supersecretkey-for-local-dev AUTH_REQUIRED=true

Start the server (runs on http://localhost:7000):
python run.py

3. Frontend Setup (React)

# Navigate to frontend cd frontend # Install dependencies (we use Yarn) yarn install

Create a .env file inside the frontend folder:

REACT_APP_API_URL=http://localhost:7000/api

Start the development app (runs on http://localhost:3000):
yarn start
